% ABSTRACT
% =============================================================================
\begin{abstract}
Material takeoff---extracting quantities from architectural plans---remains a critical bottleneck in construction estimation, consuming 50--80\% of estimator time and producing 10--15\% error rates.
This proposal describes Construction.AI, a system that automates plan-to-procurement workflows by combining a construction-domain Knowledge Graph with specialized LLM agents.
The system reads architectural plans and produces (1) an accurate bill of materials with full provenance tracking,
(2) optimized cut lists that reduce lumber waste from typical 15--20\% to under 5\%,
and (3) code-compliant build instructions with IRC/IBC citations.
The system also generates multiple structural hypotheses representing plausible load paths, evaluates candidate headers using Euler-Bernoulli beam theory with uncertainty quantification, and ranks solutions by cost, robustness, and design flexibility.
The Knowledge Graph stores structured information about components, materials, fasteners, building rules, jurisdiction-specific regulations, and historical project outcomes, enabling auditable decisions and continuous improvement.
Multi-agent collaboration separates reasoning (component inference, code validation) from deterministic optimization (cut-stock solver), achieving 95\%+ accuracy with 70--80\% time reduction compared to manual takeoff.
\end{abstract}
